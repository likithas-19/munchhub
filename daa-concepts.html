1. Trie Data Structure
Search Autocomplete

Concept: A multi-way search tree where nodes represent key characters. It enables fast prefix indexing.

Application: Powering the instant food menu search bar. As the student types letters, autocomplete matches are retrieved in $O(L)$ time where $L$ is query string length, bypassing slow databases.

2. 0/1 Knapsack (DP)
Budget Combiner

Concept: Dynamic Programming approach that solves sub-problems recursively and stores values to avoid recomputation.

Application: Powers the Budget Optimizer. It parses the prices and ratings of all canteen dishes, building the mathematically absolute best food combo within the student's cash balance.

3. Heap & Merge Sort
Trending Lists

Concept: Binary Tree-based Heap sort and Divide & Conquer-based Merge sort both solve array organization with $O(n \log n)$ bounds.

Application: Powering the sorting dropdowns inside the Trending Foods page. Students can select Popularity or Price and watch ranks adjust dynamically.

4. Queue (FIFO)
Order Pipeline

Concept: A sequential First-In-First-Out data structure where elements are inserted at the tail and removed from the head.

Application: Manages order preparations in canteens. Incoming student checkouts are queued automatically. Canteen staff fulfill orders in strict temporal checkout sequence.

5. Dijkstra's Algorithm
Shortest Delivery Path

Concept: Single-source shortest path solver on non-negative weighted graphs using a greedy search approach.

Application: Traces delivery courier paths from restaurant coordinates to hostel blocks, optimizing delivery speed and runner assignment.

6. Minimum Spanning Tree
Group Orders Pickup

Concept: Finds a subset of edges that connects all vertices in a weighted graph without cycles, while minimizing total edge weight.

Application: Solves optimal central campus pickup locations for collaborative group orders, minimizing walking distance for everyone.
